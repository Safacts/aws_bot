6 Advantages of Cloud Computing MEMORIZE

  • Trade CapEx for OpEx — No upfront server purchases, pay monthly
  • Economies of Scale — AWS buys in bulk, prices dropped 75+ times
  • Stop Guessing Capacity — Auto Scaling handles demand spikes
  • Increase Speed & Agility — Deploy in minutes instead of weeks
  • Stop Spending on Data Centers — Focus on business, not hardware
  • Go Global in Minutes — Deploy in any Region worldwide instantly

Cloud Deployment Models

  • Public Cloud — Everything on AWS. No on-premises. (startups, most companies)
  • Private Cloud — Your own data center, full control. (governments, banks with strict compliance)
  • Hybrid Cloud — Mix of AWS + on-premises connected together. MOST COMMON exam scenario

Service Models Common Trap

  • IaaS — You manage OS, apps, data. AWS manages hardware. Example: EC2
  • PaaS — You manage app and data only. Example: Elastic Beanstalk, RDS
  • SaaS — You manage nothing technical. Example: Gmail, Salesforce, Office 365
  • EC2 = IaaS · RDS = PaaS · Gmail = SaaS — know these cold

Well-Architected Framework — 6 Pillars

  • Operational Excellence — Automate operations, IaC, improve processes
  • Security — IAM, encryption, detective controls, incident response
  • Reliability — Fault tolerance, backup/recovery, handle failures
  • Performance Efficiency — Right resource types for the workload
  • Cost Optimization — Avoid waste, right pricing model, track spending
  • Sustainability (6th pillar, added 2021) — Environmental impact

Global Infrastructure

  • Regions — Separate geographic areas (e.g., us-east-1). Data stays in region unless you move it
  • Availability Zones — 1+ data centers in a Region. Minimum 3 AZs per Region. Deploy across 2+ for HA
  • Edge Locations — CloudFront CDN cache nodes. Hundreds worldwide. Deliver content with low latency
  • Local Zones — AWS infra closer to specific cities for ultra-low latency
  • Outposts — AWS rack in YOUR data center. Hybrid use case

Shared Responsibility Model 10+ Questions

  • AWS is responsible FOR the Cloud — Physical security, hardware, hypervisor, global network, managed service engines (RDS patching)
  • Customer is responsible IN the Cloud — Data, IAM users/roles/policies, OS patches on EC2, encryption configuration, firewall rules
  • EC2: you patch the OS. RDS: AWS patches the database engine. S3: AWS manages the infrastructure, you manage permissions and encryption config

IAM — Identity & Access Management

  • Users — Individual person. Long-term credentials (username + password or access keys)
  • Groups — Collection of users with shared permissions. Best practice for teams
  • Roles — Temporary credentials for services or cross-account access. EC2 → S3 = use a Role
  • Policies — JSON documents that define Allow/Deny for actions on resources
  • New IAM users start with ZERO permissions — must explicitly grant everything
  • Root account: enable MFA, never use for daily tasks, never share
  • Principle of Least Privilege — give only what is needed and nothing more

Security Services Quick Reference

  • Shield Standard — FREE, automatic, protects all AWS customers from Layer 3/4 DDoS
  • Shield Advanced — Paid, DDoS Response Team access, financial protection
  • WAF — Blocks SQL injection, XSS, bad bots. Works at Layer 7 (HTTP/HTTPS)
  • GuardDuty — ML-based threat detection. Analyzes CloudTrail, VPC Flow Logs, DNS logs
  • Inspector — Scans EC2 and containers for software vulnerabilities
  • Macie — Finds PII and sensitive data in S3 using machine learning
  • KMS — Managed encryption key service. Used with S3, EBS, RDS etc.
  • CloudTrail — Logs every API call. Who did what, when, from where
  • Config — Records resource configuration changes over time
  • Security Hub — Central dashboard aggregating findings from all security services

EC2 Pricing Models 5-7 Questions

  • On-Demand — Pay by hour/second. No commitment. Most flexible, most expensive. Use for: unknown workloads, new apps
  • Reserved — 1 or 3 year commit. Up to 72% discount. Use for: predictable, steady-state 24/7 workloads
  • Spot — Up to 90% discount using spare capacity. Can be interrupted with 2-min notice. Use for: batch jobs, fault-tolerant tasks
  • Dedicated Host — Physical server just for you. For compliance/BYOL licensing. Most expensive
  • Savings Plans — More flexible than RIs. Commit to $/hour usage across any instance type/Region/service

S3 Storage Classes 5-7 Questions

  • S3 Standard — Frequently accessed data. Highest cost, highest performance
  • S3 Standard-IA — Infrequent access, multi-AZ, retrieval fee. Backup copies, disaster recovery
  • S3 One Zone-IA — Single AZ only, 20% cheaper than Standard-IA. Data that can be recreated
  • S3 Intelligent-Tiering — Auto-moves between tiers based on access patterns. Unpredictable access
  • S3 Glacier Flexible — Archive, minutes-to-hours retrieval. Regulatory archives
  • S3 Glacier Deep Archive — Lowest cost, 12+ hour retrieval. Long-term compliance archives
  • S3 durability: 11 nines (99.999999999%) for all classes except One Zone-IA

Database Selection Guide

  • RDS — Managed relational DB. MySQL, PostgreSQL, Oracle, SQL Server, MariaDB
  • Aurora — AWS's own MySQL/PostgreSQL-compatible DB. 5x faster than MySQL
  • DynamoDB — Serverless NoSQL. Single-digit millisecond performance. Any scale
  • Redshift — Data warehouse for analytics (OLAP). Petabyte scale
  • ElastiCache — In-memory cache. Redis or Memcached. Speed up database reads
  • Neptune — Graph database for relationships (social networks, fraud detection)

Compute Services

  • EC2 — Virtual machines. You control the OS. IaaS
  • Lambda — Serverless functions. Event-driven. Max 15 min timeout. Always Free: 1M req/month
  • ECS — Managed Docker container orchestration
  • EKS — Managed Kubernetes service
  • Fargate — Serverless container engine. No EC2 to manage. Works with ECS and EKS
  • Elastic Beanstalk — PaaS. Upload code, it handles everything else

Free Tier Types Know All Three

  • 12-Month Free Tier — Available for 12 months from account creation. EC2 t2.micro 750hrs/month, S3 5GB, RDS 750hrs
  • Always Free — Never expires. Lambda 1M requests/month, DynamoDB 25GB, CloudFront 1TB/month
  • Free Trials — Service-specific short-term trial. Starts from first use, not account creation

Billing Tools

  • Pricing Calculator — Estimate costs BEFORE deploying. No account needed
  • Cost Explorer — Visualize and analyze PAST costs. Identify trends
  • Budgets — Set alerts for when spending exceeds threshold. Can trigger auto-actions
  • Cost & Usage Report — Most granular data. Line-item detail, export to S3
  • Trusted Advisor — Recommendations across 5 categories: Cost, Performance, Security, Fault Tolerance, Service Limits
  • Consolidated Billing — One bill for all accounts in AWS Organizations. Share Reserved Instance discounts

Services People Confuse Exam Traps

Service A Service B Key Difference
CloudWatch CloudTrail CloudWatch = performance metrics/alarms. CloudTrail = API audit log (who did what)
Security Group NACL SG = stateful, instance-level. NACL = stateless, subnet-level
IAM Role IAM User Role = temporary credentials for services. User = permanent credentials for humans
S3 EBS S3 = object storage (files, images). EBS = block storage (like a hard drive attached to EC2)
EBS EFS EBS = attached to 1 EC2 instance. EFS = shared file system, multiple instances simultaneously
GuardDuty Inspector GuardDuty = threat detection (behavioral). Inspector = vulnerability scanning (CVEs)
SNS SQS SNS = pub/sub fan-out to multiple subscribers. SQS = queue, one-to-one reliable delivery
Direct Connect VPN Direct Connect = physical dedicated line, consistent speed. VPN = encrypted tunnel over internet
Shield Standard Shield Advanced Standard = free, automatic. Advanced = paid, DDoS Response Team, financial protection
Reserved Instance Savings Plans RI = locked to instance type/Region. Savings Plans = flexible, commit to $/hr usage
Exam day strategy

The CLF-C02 exam is about pattern recognition. Learn these keyword-to-answer mappings and you'll handle most questions through elimination.

1
Keywords → Answers

cost-effective, save money→Spot / S3-IA / Reserved
serverless→Lambda / Fargate / DynamoDB
high availability→Multi-AZ / Auto Scaling / ELB
real-time processing→Kinesis / Lambda
DDoS protection→Shield (Standard=free)
web app attacks→AWS WAF
audit / who did what→CloudTrail
compliance reports→AWS Artifact
least overhead→Managed / serverless option
move petabytes→Snowball / Snowmobile

2
Elimination rules

  • If question mentions "no server management" → eliminate EC2 options
  • If "relational database" → eliminate DynamoDB (it's NoSQL)
  • If "SQL / transactions" → it's RDS or Aurora, not DynamoDB
  • If "physical data" transfer → Snowball/Snowmobile, not DataSync
  • If "free" DDoS protection → Shield Standard (not Advanced)
  • If "TAM" or "15-minute response" → Enterprise support only
  • Watch for NOT/EXCEPT/LEAST in bold — they flip the question
  • Two answers look similar → pick the more specific/appropriate one

3
Time management

  • 90 minutes ÷ 65 questions = ~1.4 min each
  • Read every word of every question first
  • Flag and skip if unsure — come back at end
  • Don't spend more than 2 minutes on one question
  • 15 questions are unscored — don't panic on hard ones
  • No penalty for wrong answers — always guess
  • Review flagged questions if time allows
  • Aim to finish first pass in 70 minutes

4
Day-before checklist

  • Memorize support plan response times
  • Review S3 storage classes and when to use each
  • Review EC2 pricing models (On-Demand vs Reserved vs Spot)
  • Know Shared Responsibility Model cold
  • Review IAM: users vs groups vs roles
  • Know which security service does what
  • Know the 3 Free Tier types
  • Get 8 hours sleep — seriously
